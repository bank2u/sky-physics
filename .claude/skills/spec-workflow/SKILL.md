# Spec Workflow

ดึงเนื้อหาฟิสิกส์จากครู แล้วร่างเป็น `topics/<id>/spec.md` ตาม format มาตรฐาน

## ขั้นตอน
1. ถามครูทีละน้อย **ครั้งละ 1 คำถาม** จนได้ครบทุก field ของ spec
2. เมื่อรู้เรื่อง + ระดับชั้นแล้ว ใช้ skill `physics-content` เสนอ:
   - ตัวแปรที่ควรปรับได้ (พร้อมช่วงค่าที่สมเหตุผลทางฟิสิกส์)
   - ค่าที่ควรแสดง realtime
   - simulation pattern ที่เหมาะ (เช็คก่อนว่ามีใน `shared/sim/` ไหม)
3. ให้ครูยืนยัน/แก้ ก่อนร่าง
4. ร่าง `topics/<id>/spec.md` ตาม `_system/spec-format.md` เป๊ะ ทุก field

## กฎ
- ห้ามร่าง spec ถ้ายังขาด field ใด field หนึ่ง — ถามจนครบก่อน
- ห้ามใส่ค่าฟิสิกส์ที่ไม่สมเหตุผล (เช่น มุมเกิน 90°)
- ม.ต้น เลี่ยงสมการซับซ้อน เน้นเห็นภาพ; ม.ปลาย ใส่สมการได้
- 1 spec = 1 ระดับชั้น
- `id` เป็น kebab-case ภาษาอังกฤษ (เช่น `projectile-motion`)

## Output
`topics/<id>/spec.md` — ถัดไปคือ skill `build-workflow`
