# Content Spec Format

ทุกเรื่องมีไฟล์ `topics/<id>/spec.md` ตาม format นี้ — spec-workflow ร่าง, build-workflow อ่านไปสร้าง page

## Template

```markdown
---
id: projectile-motion
title: การเคลื่อนที่แบบโพรเจกไทล์
level: ม.ปลาย          # ม.ต้น | ม.ปลาย
category: กลศาสตร์      # กลศาสตร์ | ไฟฟ้า-แม่เหล็ก | คลื่น-เสียง | ความร้อน | แสง | อื่นๆ
---

## แนวคิดที่จะสอน
อธิบายว่าเรื่องนี้สอนอะไร 2-3 ประโยค

## สิ่งที่อยากให้เด็กเห็น/เข้าใจ
- เห็นว่ามุมยิงมีผลต่อระยะอย่างไร
- เข้าใจว่าการเคลื่อนที่แยกเป็นแกน x, y อิสระกัน

## ตัวแปรที่ปรับได้
| ตัวแปร | ช่วงค่า | หน่วย | ค่าเริ่มต้น |
|--------|---------|-------|-------------|
| มุมยิง | 0–90 | องศา | 45 |
| ความเร็วต้น | 1–50 | m/s | 20 |

## ค่าที่แสดง realtime
- ระยะทางในแนวราบ (m)
- ความสูงสูงสุด (m)
- เวลาในอากาศ (s)

## สมการที่เกี่ยวข้อง
- ระยะ R = (v² sin2θ) / g

## simulation pattern ที่ใช้
projectile          # ชี้ไปที่ shared/sim/<pattern>.js

## คำอธิบายแนวคิด (ขยายความ)
ข้อความที่จะแสดงในส่วนล่างของ page
```

## กฎ
- ทุก field ต้องมี ห้ามข้าม
- `id` เป็น kebab-case ภาษาอังกฤษ
- `level` เป็น field เดียว (1 spec = 1 ระดับชั้น)
- ม.ต้น ส่วน "สมการที่เกี่ยวข้อง" อาจเว้นว่างหรือใส่แบบง่าย
- `simulation pattern ที่ใช้` ชี้ไปคลัง shared/sim/ — ถ้ายังไม่มี build-workflow จะสร้างเพิ่ม
